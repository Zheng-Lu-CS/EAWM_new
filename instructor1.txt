## 1. setup_easimulus_atari_env.sh 要求

功能：
- 从任意位置运行都能定位项目根目录；默认项目根目录是 `/data/share/hxd/zhenglu/eawm`，也允许通过 `PROJECT_ROOT=...` 覆盖。
- 进入 `$PROJECT_ROOT/EASimulus`。
- 所有输出写入 `$PROJECT_ROOT/logs/setup_easimulus_atari_<timestamp>.log`。
- 如果当前用户是 root 且系统是 Ubuntu，则优先把 apt 源切到清华源，并安装系统包：
  `ffmpeg libsm6 libxext6 libgl1 libegl1 libgl1-mesa-dev libosmesa6 libosmesa6-dev libglew-dev x11-xserver-utils xvfb mesa-utils zip rsync bison flex htop tmux`
  注意 Ubuntu 24.04 不一定有 `libgl1-mesa-glx`，不要强依赖这个旧包名。
- 如果不是 root，不要直接失败；打印缺失依赖提示。
- 创建或复用 conda env `zhenglu_easimulus`，Python 3.10。
- 配置 pip 使用清华 PyPI：
  `https://pypi.tuna.tsinghua.edu.cn/simple`
  但 PyTorch 使用官方 CUDA wheel index。
- 安装 PyTorch 2.5.1 CUDA 12.4：
  `torch==2.5.1 torchvision==0.20.1 torchaudio==2.5.1 --index-url https://download.pytorch.org/whl/cu124`
  因为服务器 driver 550.90.07 满足 CUDA 12.4 wheel 要求。
- 安装 EASimulus requirements。
- 安装 Atari ROM 所需包并导入 ROM：
  `pip install "autorom[accept-rom-license]"`
  然后执行 `AutoROM --accept-license`。
  在脚本里明确提示：运行该命令表示用户确认拥有/接受 Atari ROM 研究使用所需许可。
- 运行 `python get_lpips.py` 下载 LPIPS/VGG 附加权重到官方默认位置。
- 设置缓存目录到项目路径：
  `$PROJECT_ROOT/cache/pip`
  `$PROJECT_ROOT/cache/torch`
  `$PROJECT_ROOT/cache/huggingface`
  `$PROJECT_ROOT/cache/xdg`
  `$PROJECT_ROOT/cache/matplotlib`
- 做单卡 smoke import：
  - import torch / torchvision / gymnasium / ale_py / cv2 / hydra / omegaconf / wandb / numpy / PIL。
  - torch.cuda.is_available() 应为 True。
  - 能创建 `BreakoutNoFrameskip-v4`，reset/step 一次。
  - 能 import `src/main.py` 依赖链的关键模块，或至少运行一个非常小的官方 `python src/main.py ...` smoke 命令。
- 如果 `BreakoutNoFrameskip-v4` 创建失败但 `ALE/Breakout-v5` 成功，要做最小兼容性处理：优先确认是否需要在 `EASimulus/src/envs/wrappers/atari.py` 中 `import ale_py` 后调用 `gymnasium.register_envs(ale_py)`。只允许做这种注册兼容 patch，不要更换官方 env id 到 v5，因为复现实验应使用官方脚本中的 NoFrameskip-v4 风格。
- 最后打印如何运行 4 卡 smoke 和全量训练。

## 2. run_easimulus_atari_4gpu_smoke.sh 要求

功能：
- 从任意位置运行都能定位项目根目录；默认 `/data/share/hxd/zhenglu/eawm`。
- 激活 `zhenglu_easimulus`。
- 假设当前任务已经有 4 张 GPU 可见；检查 `torch.cuda.device_count() >= 4`。如果只有 1 卡，要明确报错，因为这是 4 卡 smoke 脚本，不是配置环境脚本。
- 对四个任务并行启动 4 个进程，每个进程只看一张 GPU：
  - GPU 0: BreakoutNoFrameskip-v4
  - GPU 1: BoxingNoFrameskip-v4
  - GPU 2: SeaquestNoFrameskip-v4
  - GPU 3: RoadRunnerNoFrameskip-v4
- 使用官方入口 `python src/main.py`，但用 smoke override 缩短运行：
  - `common.epochs=3`
  - `collection.train.stop_after_epochs=3`
  - `collection.train.config.num_steps=200`
  - `training.tokenizer.start_after_epochs=0`
  - `training.tokenizer.steps_per_epoch=1`
  - `training.world_model.start_after_epochs=1`
  - `training.world_model.steps_per_epoch=1`
  - `training.actor_critic.real_start_after_epochs=1`
  - `training.actor_critic.start_after_epochs=2`
  - `training.actor_critic.steps_per_epoch=1`
  - `evaluation.every=1`
  - `collection.test.config.num_episodes=1`
  - `collection.test.config.num_episodes_end=1`
  - `env.test.max_episode_steps=1000`
  - `common.do_checkpoint=False`
  - `wandb.mode=disabled`
- 关闭媒体/可视化输出：如果 Hydra 能解析，设置
  `collection.train.num_episodes_to_save=0`
  `collection.test.num_episodes_to_save=0`
  `evaluation.tokenizer.save_reconstructions=False`
  如果 0 导致官方代码异常，则退回为 1，但要在 README 中说明。
- 每个进程日志写入：
  `$PROJECT_ROOT/logs/smoke_easimulus_atari_<game>_<timestamp>.log`
- 等待四个进程结束；任一失败则主脚本 exit 非 0，并打印失败 log 路径。
- 设置：
  `PYTHONUNBUFFERED=1`
  `HYDRA_FULL_ERROR=1`
  `WANDB_MODE=disabled`
  `OMP_NUM_THREADS=8`
  `MKL_NUM_THREADS=8`
  `OPENBLAS_NUM_THREADS=8`
  `NUMEXPR_NUM_THREADS=8`

## 3. run_easimulus_atari_4gpu_train.sh 要求

功能：
- 和 smoke 脚本一样定位项目、激活环境、检查 4 GPU。
- 并行启动四个全量训练进程，每个任务独占一张 GPU：
  - GPU 0: BreakoutNoFrameskip-v4
  - GPU 1: BoxingNoFrameskip-v4
  - GPU 2: SeaquestNoFrameskip-v4
  - GPU 3: RoadRunnerNoFrameskip-v4
- 使用官方入口和官方核心超参。不要覆盖：
  - `common.epochs`
  - `collection.train.stop_after_epochs`
  - `collection.train.config.num_steps`
  - `training.*.steps_per_epoch`
  - `evaluation.every`
  - world model / actor critic 结构参数
- 必须使用官方 EASimulus 开关：
  `tokenizer.image.with_lpips=True`
  `benchmark=atari`
  `world_model.event_pred=True`
  `world_model.ges=True`
- `wandb.mode` 默认 `offline`，允许用户用环境变量覆盖：
  `WANDB_MODE=online bash tools/zhenglu/run_easimulus_atari_4gpu_train.sh`
- seed 默认 0，允许 `SEED=1` 覆盖。
- 输出目录放到：
  `$PROJECT_ROOT/outputs/easimulus_atari_4gpu_<timestamp>/`
- 每个进程日志写入：
  `$PROJECT_ROOT/logs/train_easimulus_atari_<game>_seed<seed>_<timestamp>.log`
- 为缓解 I/O：
  - cache 全部放 `$PROJECT_ROOT/cache`
  - 4 个任务 output dir 不能相同
  - 默认不保存媒体/可视化：
    `collection.train.num_episodes_to_save=0`
    `collection.test.num_episodes_to_save=0`
    `evaluation.tokenizer.save_reconstructions=False`
    如果 0 不兼容，退回为 1。
- 保留 checkpoint，不能关闭 `common.do_checkpoint`，因为全量训练需要可恢复。
- 进程失败时主脚本 exit 非 0，并汇总失败任务和 log 路径。
